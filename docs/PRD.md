## 二、背景與目標（Background & Goals）

### 問題陳述

傳統宮廟抽籤流程高度依賴人工，存在以下痛點：

- **信眾體驗不連續**：抽到的籤詩無法隨時查閱或回顧，容易遺忘。
- **廟方管理困難**：香油錢（捐款）以紙本或口頭登記，難以稽核與追蹤。
- **資訊不透明**：信眾無法即時查詢捐款是否成功記錄，降低信任感。
- **數位化缺口**：多數廟宇缺乏線上平台，無法服務無法親臨的信眾。

### 商業目標

1. 提供信眾隨時隨地「線上抽籤」的便利體驗。
2. 建立廟方的**電子捐款紀錄系統**，提升財務透明度與管理效率。
3. 增加廟宇的數位觸及，吸引年輕世代信眾。

### 成功指標（KPI）

| 指標                         | 目標值              |
| ---------------------------- | ------------------- |
| MVP 上線後 3 個月 MAU        | ≥ 500 位            |
| 每月線上抽籤次數             | ≥ 1,000 次          |
| 香油錢線上捐款筆數（月）     | ≥ 200 筆            |
| 用戶對抽籤結果滿意度（NPS）  | ≥ 40 分             |
| 廟方管理員紀錄查詢時間縮短   | 較紙本作業縮短 70%  |

## 三、目標使用者（Target Users）

### 用戶畫像（Persona）

#### Persona A — 一般信眾（主要用戶）
- **年齡**：20–65 歲
- **特徵**：有拜拜習慣，偶爾或定期到廟裡參拜，希望方便快速獲得神明指引
- **痛點**：無法常常親臨廟宇；抽到的籤詩看不懂、忘記帶走
- **需求**：隨時查詢籤詩解釋、回顧歷史抽籤紀錄、簡單捐款

#### Persona B — 廟方管理員（後台用戶）
- **年齡**：35–60 歲
- **特徵**：負責廟務行政，對資訊系統操作能力有限
- **痛點**：手動統計香油錢耗時費力，帳目難以對帳
- **需求**：查看捐款紀錄、匯出報表、管理籤詩內容

---

### 用戶旅程（User Journey）

**信眾抽籤流程：**
```
進入網站 → 選擇神明 → 誠心祈願（輸入心願） → 搖籤 → 顯示籤號與籤詩
→ 查看籤詩解釋與建議 → 抽籤紀錄自動儲存 → 可選擇捐贈香油錢
```

**信眾捐款流程：**
```
進入捐款頁 → 輸入金額與姓名 → 選擇捐款方式（模擬） → 送出 → 收到感謝通知
→ 可在捐款紀錄頁查詢歷史
```

### 主要使用場景

- 信眾在家或外出時，透過手機瀏覽器隨時抽籤
- 廟方人員每日登入後台查看當日捐款匯總
- 信眾在廟裡現場掃 QR Code 進入系統完成線上抽籤與捐款

### 模組一：抽籤系統

#### FR-01 ｜ 虛擬搖籤（P0）

> 身為**信眾**，我想要**在線上進行虛擬搖籤**，以便**隨時隨地獲得神明指引**。

- 系統以動畫模擬搖籤過程（籤筒搖動效果）
- 籤號從 1–60 號隨機抽出（可由廟方設定範圍）
- 抽出後顯示籤號、籤詩全文、籤詩等級（大吉、中吉、小吉、凶）

**驗收條件：**
- [ ] 點擊「搖籤」後有動畫效果，持續 1–3 秒
- [ ] 顯示籤號（如：第 32 籤）
- [ ] 顯示籤詩全文（4 句漢字）
- [ ] 顯示吉凶等級標籤
- [ ] 顯示籤詩白話解釋

---

#### FR-02 ｜ 籤詩解釋（P0）

> 身為**信眾**，我想要**看到籤詩的白話說明**，以便**理解神明的指示**。

- 每支籤對應預設白話解釋（婚姻、事業、健康、學業四面向）
- 解釋文字清楚易懂，避免艱澀用語

**驗收條件：**
- [ ] 每籤均有對應白話解釋
- [ ] 解釋按面向分類顯示（可展開/收合）

---

#### FR-03 ｜ 抽籤歷史紀錄（P0）

> 身為**信眾**，我想要**查詢自己的抽籤歷史**，以便**回顧過去的指引**。

- 每次抽籤自動儲存紀錄（含：日期時間、籤號、神明、心願文字）
- 提供列表顯示，最近一次在最上方
- 支援依日期篩選

**驗收條件：**
- [ ] 抽籤後 5 秒內紀錄寫入資料庫
- [ ] 歷史頁顯示：日期、時間、籤號、吉凶等級
- [ ] 點擊可展開查看籤詩全文與解釋
- [ ] 預設顯示最近 20 筆，支援分頁

---

#### FR-04 ｜ 神明選擇（P1）

> 身為**信眾**，我想要**選擇要請示的神明**，以便**針對不同事務求籤**。

- 提供至少 3 位預設神明（如：媽祖、關聖帝君、城隍爺）
- 不同神明可對應不同籤詩組

**驗收條件：**
- [ ] 首頁顯示神明選擇介面
- [ ] 選擇神明後進入對應籤詩庫

---

### 模組二：香油錢捐贈系統

#### FR-05 ｜ 香油錢捐贈表單（P0）

> 身為**信眾**，我想要**線上捐贈香油錢**，以便**表達對神明的虔誠心意**。

- 提供固定金額選項（100、300、500、1000 元）及自訂金額輸入
- 填寫資訊：姓名（選填）、聯絡電話（選填）、心願備註（選填）
- 送出後顯示感謝畫面與流水號

**驗收條件：**
- [ ] 金額為必填，最低 1 元
- [ ] 自訂金額只接受正整數
- [ ] 送出後顯示「感謝 XX，捐款 $XXX 已成功記錄」
- [ ] 顯示流水號（唯一識別碼）供事後查詢

---

#### FR-06 ｜ 捐款歷史紀錄（P0）

> 身為**信眾**，我想要**查詢自己的捐款紀錄**，以便**確認捐款是否成功**。

- 依流水號查詢單筆捐款紀錄
- 列表顯示捐款日期、金額、流水號

**驗收條件：**
- [ ] 輸入流水號可查詢單筆紀錄
- [ ] 捐款列表顯示：日期、金額、備註（若有）
- [ ] 預設顯示最近 20 筆，支援分頁

---

### 模組三：管理後台

#### FR-07 ｜ 管理員登入（P0）

- 廟方管理員以帳號密碼登入後台
- 未登入者無法存取後台任何頁面

**驗收條件：**
- [ ] 登入失敗顯示錯誤訊息
- [ ] Session 逾時（30 分鐘未操作）自動登出

---

#### FR-08 ｜ 捐款統計與匯出（P1）

> 身為**廟方管理員**，我想要**查看捐款統計報表**，以便**掌握廟宇財務狀況**。

- 顯示當日、當週、當月捐款總額
- 列表顯示所有捐款明細，支援日期區間篩選
- 匯出為 CSV 檔案

**驗收條件：**
- [ ] 統計數字即時反映最新資料
- [ ] 篩選後匯出的 CSV 含所有欄位（流水號、日期、姓名、金額、備註）

---

#### FR-09 ｜ 抽籤統計（P2）

- 顯示各籤號被抽到次數排行
- 顯示每日抽籤總次數走勢圖

**驗收條件：**
- [ ] 排行榜顯示前 10 名最常被抽到的籤號
- [ ] 走勢圖顯示最近 30 天資料

---

#### FR-10 ｜ 籤詩內容管理（P2）

- 管理員可新增、編輯、刪除籤詩內容
- 支援各籤號的白話解釋維護

**驗收條件：**
- [ ] 編輯後即時生效（信眾下次抽籤時顯示新內容）
- [ ] 刪除前有確認提示

## 六、非功能需求（Non-Functional Requirements）

| 類別         | 需求描述                                                  |
| ------------ | --------------------------------------------------------- |
| **效能**     | 頁面首次載入 < 2 秒；抽籤 API 回應 < 300ms               |
| **安全性**   | 後台使用 Session 驗證；捐款資料不儲存金融資訊            |
| **可用性**   | 服務可用率 ≥ 99%（月計算）                                |
| **相容性**   | 支援 Chrome / Safari / Firefox 最新兩個版本；支援手機瀏覽 |
| **擴展性**   | 籤詩資料庫、捐款紀錄設計可水平擴展                        |
| **無障礙性** | 色彩對比符合 WCAG 2.1 AA；文字大小可調整                  |
| **語言支援** | 繁體中文為主，介面文字不出現簡體或英文混用                |

## 七、UX / UI 設計

### 用戶流程圖（User Flow）

```
[首頁]
  ├─→ [選擇神明]
  │       └─→ [輸入心願（選填）] → [搖籤動畫] → [顯示籤詩結果]
  │                                                   └─→ [查看歷史紀錄]
  │                                                   └─→ [前往捐款]
  └─→ [捐款頁面]
  │       └─→ [填寫金額與資料] → [送出] → [感謝頁（含流水號）]
  └─→ [歷史紀錄頁]
          ├─→ [抽籤紀錄]
          └─→ [捐款紀錄]
```

### 視覺設計方向

- **風格**：融合傳統廟宇美學（朱紅、金色）與現代簡潔 UI
- **主色系**：深紅 `#8B0000`、金黃 `#FFD700`、米白 `#FFF8E7`
- **字體**：標題使用楷書或明體字型；內文使用無襯線體提升閱讀性
- **動畫**：抽籤時有籤筒搖動動畫（CSS / JS 實作）；結果揭曉有淡入效果

### 互動邏輯說明

| 情境             | 處理方式                              |
| ---------------- | ------------------------------------- |
| 重複點擊「搖籤」 | 動畫期間禁用按鈕，防止重複觸發        |
| 捐款金額未填     | 顯示紅色 error 提示「請輸入捐款金額」 |
| 歷史紀錄為空     | 顯示 empty state 插圖與引導文字       |
| 網路中斷         | 顯示 Toast 錯誤通知，保留表單內容     |

## 八、數據需求（Data Requirements）

### 需追蹤的行為事件

| 事件名稱          | 觸發時機             | 帶入參數                        |
| ----------------- | -------------------- | ------------------------------- |
| `shake_start`     | 用戶點擊搖籤按鈕     | `god_id`, `wish_text`（脫敏）   |
| `shake_result`    | 籤詩結果顯示         | `god_id`, `fortune_no`, `level` |
| `donation_submit` | 捐款表單送出         | `amount`, `is_custom`           |
| `history_view`    | 查看歷史紀錄頁       | `tab`（fortune / donation）     |

### 個資與隱私

- 捐款時姓名、電話為**選填**，儲存前需取得用戶同意
- 系統不強制要求身份驗證，以 **Session ID** 識別同一瀏覽器的操作紀錄
- 需在首頁顯示**隱私政策聲明**，符合個資法規範

## 十、風險與假設（Risks & Assumptions）

### 假設

1. 廟方已備妥完整的籤詩資料（1–60 號籤詩、白話解釋）
2. MVP 以單一廟宇為範圍，不需多租戶架構
3. 捐款為紀錄性質，暫不串接真實金流（無實際收款）

### 風險

| 風險                           | 影響 | 緩解策略                                    |
| ------------------------------ | ---- | ------------------------------------------- |
| 籤詩內容著作權爭議             | 中   | 由廟方提供或自行撰寫白話解釋                |
| 信眾誤以為線上捐款等同真實入帳 | 高   | 介面清楚標示「此為意願記錄，非實際金流」    |
| 管理員資安意識不足導致帳密洩露 | 高   | 強制密碼規則 + 定期提醒更換密碼             |
| 手機瀏覽器相容性問題           | 低   | 採用 Vanilla JS，避免複雜依賴               |
